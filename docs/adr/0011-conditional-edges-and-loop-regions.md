# Conditionnels sur les edges + boucles matérialisées nommées (suppression de Switch / Loop / ForEach)

Les nœuds `Switch`, `Loop` et `ForEach` étaient des nœuds first-class portant la logique de routage et d'itération (cf. ADR-0002 pour le compromis « conditionnels mécaniques portés par un nœud routeur dédié »). À l'usage, ils créaient une cérémonie et une confusion réelles : un `Switch` n'est qu'un pass-through qui ré-émet son input sur un port gardé — un hop fantôme dans le dataflow qui rendait déjà le passage des outputs au nœud suivant difficile à lire. Une session `/prototype` (variant C, 2026-06-05) a comparé quatre directions et retenu « conditions sur l'edge + boucle comme région ».

**Décision.** On supprime les nœuds `Switch`, `Loop` et `ForEach`.

(a) **Le routage conditionnel vit sur l'edge.** Chaque edge porte une clause `when:` optionnelle, évaluée en **multi-match** : toutes les edges satisfaites quittant un même output port firent (fan-out possible) ; pas de `first-match-wins` ; une edge `else` fire si aucune edge sœur n'a matché. `iter` redevient un champ référençable dans `when:` (il n'avait été retiré que parce que le nœud `Loop` portait le compteur). La grammaire de prédicats est inchangée (`eq`/`neq`/`lt`/`lte`/`gt`/`gte`/`in`/`not_in`, jamais d'eval libre ni de LLM-router).

(b) **Les boucles sont des entrées nommées du bloc `loops:`** du YAML : `id` + `members` explicites (≥ 1 nœud) + `kind: bounded | collection`, avec un compteur d'itération **par-boucle keyé sur l'id**. Une boucle `bounded` (compteur, `max_iter`) **naît par auto-détection** d'un cycle — self-edge incluse — pour qu'aucun cycle ne soit accidentellement non-borné ; une boucle `collection` (ex-ForEach, driver `over: <field>`) **naît par geste explicite**, faute de signature topologique. **Aucune edge n'est marquée « back-edge »** : son rôle est dérivé de la boucle. L'épuisement d'une boucle `bounded` route via une sortie conditionnée `iter >= max`, ou bloque explicitement (« exhausted — unrouted », routable par le Pipeline Manager) — jamais de stall silencieux. La sortie d'une boucle `collection` est une **barrière** (tous items terminés → une seule fire), préservant `done → Merge` (ADR-0006). « Région » désigne le rendu canvas ; le concept modèle est la boucle nommée.

**Pourquoi.** Contre *garder les nœuds et n'en changer que le rendu* : l'indirection du Switch pass-through et la confusion sur le passage des outputs **étaient** le coût ; un rendu ne purge pas la dette. Contre *boucle auto-détectée sans identité (variant C pur)* : « quelle edge est la back-edge » est une propriété topologique globale qui bascule dès qu'on édite ailleurs ; une **identité nommée** sort la boucle de la topologie, stabilise la persistance du bound (sur la boucle, pas sur une edge fragile), et ouvre les boucles imbriquées (sémantique d'itération composite **différée** : v1 = itération plate). Contre *first-match-wins* : exigerait de surfacer un ordre de priorité sur le canvas (variant C n'en montre pas) — un routage à précédence invisible est un footgun ; le multi-match n'a rien à cacher. Risques acceptés : le multi-match peut fan-out sur conditions chevauchantes (assumé, ADR-0001 *sharp tool* ; converger via `Merge` au besoin) ; un cycle dessiné hors d'une boucle serait non-borné (mitigé par l'auto-matérialisation `bounded`).

Supersede le placement de la condition décidé en **ADR-0002** (qui plaçait `when:` sur l'edge, puis fut migré vers les ports du Switch) ; le principe d'ADR-0002 — conditions mécaniques, jamais de LLM-router — **tient**. Interagit avec ADR-0006 (Merge reste un nœud first-class, cible de convergence des `collection`) et ADR-0007 (édition pendant un Run : nœuds running immuables, edges libres ; détruire une boucle dont les membres itèrent est régi par cette règle).
